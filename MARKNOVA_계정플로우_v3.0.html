  • 로그인 실패 제한 제외 결정 (2026-03-18 회의)

계정 관리 기능

  • 개인정보 수정 이름, 전화번호, 부서 변경 (이메일/역할은 변경 불가)
  • 비밀번호 변경 현재 비밀번호 확인 후 새 비밀번호 설정
  • ID 찾기 이름 + 전화번호 입력 → 이메일(ID) 마스킹 표시 (로그인 화면, 시스템 관리자 제외 전 역할 공통)
  • 비밀번호 초기화 상위 담당자에게 요청 → 비밀번호 초기화 → 이메일 링크 재발송 → 본인이 새 비밀번호 설정
  • 비밀번호 초기화 경로
    · 시스템 관리자 → 개발팀 DB 직접 초기화
    · 마크노바 CS 담당자 → 시스템 관리자에게 요청
    · 기관관리자 → 마크노바 CS 담당자에게 요청
    · 운영자 → 기관관리자에게 요청
    · 응급콜센터 직원 → 마크노바 CS 담당자에게 요청
  • 마크노바 시스템 관리자 본인 비밀번호 변경 가능 (1인 1계정, 최초 로그인 시 강제 변경 아님)

계정 생성 / 이메일 링크

  • 계정 생성 정책 모든 역할 공통 — 생성 직후 비활성 상태 (비밀번호 없음), 비밀번호 설정 완료 전까지 로그인 불가. 임시 비밀번호 없이 이메일 링크가 인증 역할을 대체
  • 마크노바 CS 담당자 마크노바 시스템 관리자가 이름/이메일/전화번호 입력 → 비활성 계정 생성 → 이메일 링크 발송 → 비밀번호 설정 → 약관 동의 → 활성화
  • 기관관리자 계정 기관 생성 시 담당자 비활성 계정 함께 생성 → 이메일 링크 자동 발송
  • 운영자 계정 생성 담당자가 이름/ID(이메일)/전화번호 입력 → 비활성 계정 생성 → 이메일 링크 자동 발송 (별도 회원가입 없음)
  • 링크 유효기간 발송일로부터 5일 — 5일 내 재접속 가능, 만료 시 새 링크 재전송

  • 링크 클릭 후 (기관관리자) 비밀번호 설정 → 약관 동의 → 계정 활성화
  • 링크 클릭 후 (응급콜센터 직원) 비밀번호 설정 → 약관 동의 → 계정 활성화
