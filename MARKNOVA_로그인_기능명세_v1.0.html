검증 규칙
4-1. 프론트 검증 (버튼 클릭 시, 서버 호출 전)
필드검증 규칙실패 시 에러
이메일 + 비밀번호 둘 다 비어있는지 확인 이메일과 비밀번호를 입력해주세요.
이메일 이메일 형식 확인 (예: name@domain.com) 올바른 이메일 형식을 입력해주세요.
비밀번호 영문(대소문자) 최소 1자 + 숫자 최소 1자 + 특수문자 최소 1자 포함, 총 8자 이상
특수문자 필수 (개인정보보호위원회 규정)

허용 예시: Abc1234! ✅ / Mark1234! ✅
불가 예시: abc12345 ❌ (특수문자 없음) / abcdefgh ❌ (숫자 없음) / 12345678 ❌ (영문 없음) / abc123 ❌ (8자 미만)		 비밀번호는 영문, 숫자, 특수문자 조합 8자 이상이어야 합니다.
검증 순서: ① 둘 다 미입력 → ② 이메일 형식 → ③ 비밀번호 규칙 → 모두 통과 시 서버 인증
에러는 첫 번째 실패한 것 1개만 표시. 이전 에러가 있으면 입력값 변경 시 자동으로 사라짐.
4-2. 서버 인증 (프론트 검증 통과 후)
검증내용실패 시 에러
이메일 + 비밀번호 매칭 입력한 이메일과 비밀번호가 일치하는 계정이 있는지 확인 이메일 또는 비밀번호가 올바르지 않습니다.
계정 활성화 여부 이메일/비밀번호는 맞지만 아직 비밀번호를 설정하지 않은 계정 (이메일 링크 미클릭) 계정이 활성화되지 않았습니다. 이메일로 발송된 활성화 링크를 확인해주세요.

로그인 플로우차트

로그인 플로우 요약

  1. 이메일 + 비밀번호 입력
  2. "로그인" 버튼 클릭
  3. 프론트 검증: 미입력 체크 → 이메일 형식 → 비밀번호 규칙
  4. 서버 인증: 이메일+비밀번호 매칭 + 계정 활성화 여부 확인
  5. 성공 → 역할별 대시보드 진입
  6. 실패 → 에러 메시지 표시 → 재입력

관련 정책
정책내용
통합 로그인 모든 역할이 동일한 URL로 접속. 로그인 성공 시 역할에 따라 대시보드가 달라짐. 역할별 별도 URL 없음.
비밀번호 규칙 영문 + 숫자 + 특수문자 조합 8자 이상 필수 (개인정보보호위원회 규정).
비활성 계정 이메일 링크 미클릭(비밀번호 미설정) 계정은 로그인 불가. 활성화 완료 후 로그인 가능.
